Project Based – Project UAS Mobile Programming (2024)

Deskripsi:
Project ini merupakan pengembangan aplikasi berbasis mobile yang dikerjakan sebagai tugas Ujian Akhir Semester pada mata kuliah Mobile Programming. Aplikasi dikembangkan menggunakan bahasa pemrograman Python dan framework Flutter. Dalam project ini, saya berperan sebagai pemrogram yang bertanggung jawab mengintegrasikan desain antarmuka yang dibuat oleh rekan tim dengan API yang telah disediakan oleh dosen, sehingga data dapat ditampilkan secara dinamis sesuai kebutuhan aplikasi. Aplikasi ini dirancang untuk menampilkan data lagu, detail informasi, serta fitur pengelolaan favorit sebagai bagian dari fungsionalitas utama.


Refleksi:
What: Melalui pengembangan project ini, saya terlibat langsung dalam pembangunan aplikasi berbasis mobile menggunakan bahasa pemrograman Python dan framework Flutter, khususnya pada proses integrasi desain antarmuka dengan API yang menyediakan data aplikasi. Saya bertanggung jawab memastikan data dapat diambil, diolah, dan ditampilkan secara dinamis sesuai kebutuhan fungsional aplikasi.
So What: Dari proses tersebut, saya memperoleh pemahaman bahwa pengembangan aplikasi mobile tidak hanya berfokus pada tampilan antarmuka, tetapi juga pada integrasi antara frontend dan backend. Saya mempelajari mekanisme pemanggilan API, pengolahan data berformat JSON, serta pengelolaan state pada Flutter agar aplikasi berjalan secara konsisten dan responsif. Pengalaman ini juga memperkuat pemahaman saya mengenai pentingnya struktur kode yang terorganisasi dan koordinasi tim dalam pengembangan perangkat lunak.
Now What: Berdasarkan pembelajaran yang diperoleh, saya berencana menerapkan konsep integrasi API, pengelolaan state, dan struktur aplikasi yang lebih modular pada pengembangan project mobile selanjutnya. Selain itu, aplikasi ini akan dikembangkan lebih lanjut melalui penambahan fitur dan optimasi performa sebagai bekal teknis dalam menghadapi pengembangan aplikasi mobile di dunia kerja profesional.

Case Based – Sistem Informasi PORSENIJAR Kota Denpasar Berbasis WEB (TA)

Deskripsi:
Studi kasus ini membahas pengelolaan kegiatan Pekan Olahraga dan Seni Pelajar (PORSENIJAR) Kota Denpasar yang sebelumnya masih dilakukan menggunakan aplikasi umum seperti Google Form dan Google Spreadsheet. Kondisi tersebut menimbulkan permasalahan berupa duplikasi data, risiko terhapusnya data, serta keterlambatan dalam penyampaian informasi perolehan medali. Untuk mengatasi permasalahan tersebut, dikembangkan sistem informasi berbasis web dengan menerapkan konsep Role-Based Access Control (RBAC) menggunakan metode pengembangan Prototype, sehingga setiap pengguna memiliki hak akses sesuai dengan perannya.


Refleksi:
What:Saya mempelajari proses perancangan dan pengembangan sistem informasi berbasis web untuk pengelolaan PORSENIJAR dengan fokus pada penerapan konsep Role-Based Access Control (RBAC). Proses ini mencakup analisis permasalahan sistem lama, perancangan kebutuhan pengguna, serta pemahaman alur kerja setiap peran dalam sistem.
So What:Melalui studi kasus ini, saya memahami bahwa keamanan sistem tidak hanya berkaitan dengan perlindungan data, tetapi juga pengaturan hak akses yang tepat. Penerapan RBAC membantu mencegah kesalahan pengelolaan data dan meningkatkan keandalan sistem. Selain itu, metode Prototype memungkinkan penyesuaian sistem berdasarkan umpan balik pengguna secara bertahap.
Now What:Ke depannya, saya akan menerapkan konsep RBAC, analisis kebutuhan pengguna, dan perancangan sistem yang terstruktur dalam pengembangan sistem informasi lainnya. Pembelajaran ini menjadi bekal untuk membangun sistem yang lebih aman, terorganisasi, dan siap diterapkan di lingkungan kerja profesional.
